# iLogCat - 需求分析

## 项目概述

**项目名称：** iLogCat
**目标平台：** Linux (主要支持 Ubuntu)
**参考产品：** Android Studio Logcat

## 核心功能需求

### 1. 日志级别彩色显示

| 级别 | 颜色 | 说明 |
|------|------|------|
| VERBOSE | 灰色 | 详细日志 |
| DEBUG | 蓝色 | 调试日志 |
| INFO | 绿色 | 信息日志 |
| WARN | 橙色 | 警告日志 |
| ERROR | 红色 | 错误日志 |
| FATAL | 红色加粗 | 致命错误 |

### 2. 关键字过滤

**单关键字过滤：**
- 输入关键字，高亮匹配行
- 支持正则表达式
- 大小写敏感/不敏感切换

**多关键字过滤：**
- 支持同时过滤多个关键字
- AND/OR 逻辑组合
- 每个关键字独立颜色高亮

### 3. 日志来源

**本地日志：**
- 执行命令获取日志（如 `dmesg`、`journalctl`）
- 跟踪日志文件实时刷新（`tail -f` 效果）
- 查看系统完整日志

**远程日志（SSH）：**
- 连接 SSH 服务器
- 执行远程命令获取日志
- 跟踪远程日志文件
- 所有过滤功能与本地一致

## 用户故事

### US-001: 查看本地系统日志
> 作为开发者，我希望能够查看本地系统日志，以便排查系统问题。

**验收标准：**
- [ ] 能够执行 `dmesg` 命令并显示日志
- [ ] 能够执行 `journalctl` 命令并显示日志
- [ ] 日志按级别显示不同颜色
- [ ] 能够滚动查看历史日志

### US-002: 实时跟踪日志文件
> 作为开发者，我希望能够实时跟踪日志文件的变化，以便及时发现问题。

**验收标准：**
- [ ] 能够打开并跟踪指定日志文件
- [ ] 新日志自动追加显示
- [ ] 能够暂停/恢复跟踪
- [ ] 能够清空当前显示

### US-003: 关键字过滤
> 作为开发者，我希望能够通过关键字过滤日志，以便快速定位相关信息。

**验收标准：**
- [ ] 能够输入单个关键字过滤
- [ ] 支持正则表达式
- [ ] 匹配行高亮显示
- [ ] 能够添加多个过滤条件
- [ ] 支持 AND/OR 组合

### US-004: SSH 远程日志
> 作为开发者，我希望能够通过 SSH 连接远程服务器查看日志，以便排查远程环境问题。

**验收标准：**
- [ ] 能够配置 SSH 连接（主机、端口、用户、密码/密钥）
- [ ] 能够保存多个 SSH 配置
- [ ] 连接后能够执行远程命令获取日志
- [ ] 能够跟踪远程日志文件
- [ ] 过滤功能与本地一致

### US-005: 日志级别筛选
> 作为开发者，我希望能够按日志级别筛选，以便只查看关心的日志。

**验收标准：**
- [ ] 下拉选择日志级别
- [ ] 选择后只显示该级别及以上日志
- [ ] 能够多选级别

## 非功能需求

### 性能
- 日志显示流畅，不卡顿
- 支持大量日志（10万行+）
- 搜索响应时间 < 500ms

### 可用性
- 界面直观，参考 Android Studio Logcat
- 支持键盘快捷键
- 支持配置保存/恢复

### 兼容性
- 主要支持 Ubuntu 20.04+
- 兼容其他 Linux 发行版

## 技术约束

- 使用 Rust 开发（性能 + 安全）
- GUI 使用 GTK4（Linux 原生体验）
- SSH 使用 ssh2-rs 库
- 配置文件使用 TOML 格式
